* MDD 신규추가기능
1. 원격 개발
	: 개발 환경에 접근하는 모든것을 MDD 내에서 가능하게 만드는 것이 목표

(INITCONFIG 내에는 BMSCTX 서버 IP, 사용기간이 입력되어 있다)

//모델
2. DXDBINIT 파일
	: 기존에는 접속하는 프로젝트 ID 별로 서버 접속 환경이 필요했다
	<LOCAL> 태그 아래 <DEFAULT> : 따로 환경을 잡을 필요가 없을 경우
	<LOCAL> 태그 아래 <프로젝트명> : 해당 프로젝트명에 대한 빌드 패스 등의 경로가 따로 필요할 때

	<LOGININFO>태그 > <SECURITY> : 서버와 클라이언트의 통신을 암호화

3. BOC 검색 기능
	: 첫 검색 조회시 모든 메뉴 정보를 조회해야 하기 때문에 시간이 오래 걸림
	  검색창 좌측 체크박스가 체크되어 있을 땐 조회 시간이 빠름
	  메뉴 리로드시 체크박스가 해제됌

4. TESTDB 기능
	: DB가 접근하는 대상을 임의로 설정해 테스트할 수 있음

5. BOC 실행 조건
	: 각 BOC에 설정된 실행 조건이 트루일 때만 BOC가 실행됌
	  BOC의 썸네일 글자가 빨간색인 것은 실행 조건이 설정되어 있다는 것

6. UNDO, REDO
	: 서비스에 대한 변화가 있을때 활성화

7: FTP BOC변화
	: 기존 FTP BOC는 SFTP를 지원하지 않음, FTP BOC가 삭제되고 CMD로 FTP를 연결시키는 방식으로 생성

8. 상단 메뉴바 FUNCTION
	: 직접 사용할 공통 기능들 등록해서 사용 가능, 유사한 기능들이 많아지는걸 막기 위함
	: 로컬에 떨어지는 파일이기 때문에 서로 배포할 필요가 있음

9. 스크립트 바로가기 등록
	: 화면에서 작성한 스크립트를 ALT+번호 클릭으로 등록할 수 있음.(개인)

10. 스크립트 에디터 > 로그 테일
	: SERVER ADMIN - START > START클릭 > STOP(개인)

11. 스크립트 에디터 > 서버 커맨드
	: 서버 재기동, 중지 등 조작

12. 그리드 > SUMMARY COL
	: 그냥 직접 써보고 확인하기

13. SUBMISSION > ACTION MENU 더블클릭	
	: 해당 서비스를 서비스모델에서 찾아줌

14. SQLDEVELOPER
	: 테이블명; 으로 SELECT	* FROM 테이블명 쿼리를 실행시킬 수 있음

15. DATA MODEL	
	: 데이터 딕셔너리에 유효한 값을 추가해야 사용 가능

16. PROCESSBULIDER_LOGTAIL.XML > SERVER CONFIG
	: <프로젝트명>태그 아래 <CMDLIST> 

17. PROCESSBUILDER_CONFIG > <프로젝트명>
	: <TEST_DB> 태그 추가

	: <UX-DESIGNER> 태그 밑 <OBFUSCATION>태그 추가 (압축을 위한 태그)
		jsp 빌드타입 full : 기존 빌드 방식
		partition : 스크립트 압축 후 js로 빼내는 방식
